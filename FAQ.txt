FAQ – Python Module 03: Data Quest

====================================

Este documento recoge aclaraciones conceptuales sobre el módulo
Python Module 03 – Data Quest.
Su objetivo es facilitar la comprensión del subject y apoyar la
explicación durante la evaluación.

No forma parte del código entregable.

1. ¿Qué es sys.argv?

sys.argv es una lista que contiene los argumentos pasados al programa
desde la línea de comandos.

El primer elemento (sys.argv[0]) es siempre el nombre del archivo
ejecutado.
Los elementos siguientes son los valores introducidos por la persona
usuaria al ejecutar el programa.

2. ¿Por qué sys.argv siempre tiene al menos un elemento?

Porque Python incluye automáticamente el nombre del script ejecutado
como primer argumento, incluso cuando no se pasan parámetros
adicionales.

3. ¿Cómo sabe el programa qué debe hacer?

El programa interpreta su comportamiento a partir de los argumentos
recibidos por la línea de comandos.

En este módulo, los argumentos representan directamente los datos
a procesar (puntuaciones, coordenadas, eventos, etc.).

4. ¿Por qué usar listas para procesar datos?

Las listas son adecuadas para manejar datos secuenciales porque:

mantienen el orden de entrada

permiten recorrer y procesar los elementos fácilmente

facilitan el cálculo de estadísticas básicas

Por eso son ideales para análisis simples de datos recibidos por
línea de comandos.

5. ¿Qué es enumerate() en Python?

enumerate() es una función integrada de Python que permite recorrer
una colección y obtener, al mismo tiempo:

el índice

el valor de cada elemento

En cada iteración devuelve un par (índice, valor).

6. ¿Por qué usar enumerate() en lugar de range(len())?

Porque enumerate():

mejora la legibilidad del código

evita errores relacionados con índices

expresa mejor la intención del recorrido

Comparación:

for i in range(len(scores)):
    print(i, scores[i])

for i, score in enumerate(scores):
    print(i, score)


La segunda forma es más clara y directa.

7. ¿Para qué se usa enumerate() en este módulo?

Se utiliza cuando es necesario:

numerar resultados

mostrar posiciones junto a valores

recorrer listas sin perder la referencia del índice

En ejercicios de análisis de datos, enumerate() permite relacionar
cada valor con su posición de forma explícita y ordenada.

8. ¿Por qué usar try/except en este módulo?

Porque la entrada de datos no siempre es válida.

El uso de try/except permite:

capturar errores sin que el programa se cierre

mostrar mensajes claros a la persona usuaria

mantener un flujo de ejecución controlado

Esto demuestra código robusto y consciente de errores.

9. ¿Qué significa manejar errores “sin que el programa crashee”?

Significa que, aunque ocurra un error (por ejemplo, un valor no
convertible a entero), el programa:

informa del problema

termina de forma limpia y controlada

Este comportamiento cumple el requisito de graceful handling
especificado en el módulo.

10. ¿Qué es el unpacking en Python?

El unpacking (desempaquetado) es una forma de asignar los valores de
una estructura iterable (como una tupla o una lista) a varias
variables en una sola operación.

Ejemplo con una tupla de coordenadas:

position = (3, 4, 0)
x, y, z = position


Cada elemento de la tupla se asigna automáticamente a una variable
distinta.

11. ¿Por qué el unpacking es útil en este módulo?

Porque permite trabajar con datos estructurados de forma clara y
legible.

En el ejercicio de coordenadas 3D:

las posiciones se representan como tuplas

el unpacking facilita acceder a cada componente (x, y, z)

evita accesos menos legibles como position[0], position[1], etc.

Esto mejora la claridad del código y refleja una intención explícita.

12. ¿Qué ocurre si el número de variables no coincide?

Python lanza un ValueError si el número de variables no coincide con
el número de elementos del iterable.

Ejemplo:

position = (1, 2)
x, y, z = position   # Error


Esto refuerza que el unpacking es seguro y explícito: la estructura
de los datos debe coincidir con lo que el programa espera.

13. ¿Por qué usar tuplas para coordenadas?

Las tuplas son adecuadas para representar coordenadas porque:

agrupan datos relacionados (x, y, z)

son inmutables, lo que evita modificaciones accidentales

expresan claramente una posición fija en el espacio

Esto encaja bien con el concepto de coordenadas en un sistema 3D.

14. ¿Cómo se parsean coordenadas desde un string?

Las coordenadas se reciben como strings en el formato "x,y,z".

El proceso consiste en:

separar los valores con split(",")

convertir cada valor a entero con int()

manejar posibles errores con try/except

Este enfoque garantiza que solo se trabajará con datos válidos.

15. ¿Por qué usar if __name__ == "__main__"?

Esta estructura permite:

separar la lógica del programa de su ejecución

evitar ejecuciones accidentales al importar el archivo

mejorar la claridad y reutilización del código

Aunque no es estrictamente obligatorio, demuestra una práctica
más profesional y escalable.

18. ¿Qué es la distancia euclídea (3D) y cómo se calcula?

La distancia euclídea es la distancia “en línea recta” entre dos
puntos. En 3D se calcula aplicando la extensión del teorema de
Pitágoras a los tres ejes (X, Y, Z).

Para dos puntos:

p1 = (x1, y1, z1)
p2 = (x2, y2, z2)


Desplazamientos:

Δx = x2 - x1
Δy = y2 - y1
Δz = z2 - z1


Cálculo:

distancia = sqrt(Δx² + Δy² + Δz²)


Ejemplo:

p1 = (0, 0, 0)
p2 = (3, 4, 0)

distancia = sqrt(3² + 4² + 0²) = 5


En el código, esta fórmula se implementa usando math.sqrt y
unpacking de las tuplas para acceder a x, y y z.
19. Why are tuples perfect for 3D coordinate data, and how does unpacking
make the code more readable and powerful?
--------------------------------------------------------------------------

Tuples are well suited for representing 3D coordinates because they group
related values (x, y, z) in a fixed and ordered structure.

Since tuples are immutable, once a coordinate is created it cannot be
modified accidentally, which makes them ideal for representing positions
in a coordinate system.

Unpacking further improves readability by assigning each component of the
tuple to a variable with clear semantic meaning:

    position = (3, 4, 0)
    x, y, z = position

This avoids accessing values by index (position[0], position[1], etc.) and
makes the intention of the code explicit.

By combining tuples with unpacking, the code becomes easier to read,
less error-prone, and more expressive when working with structured data
such as 3D coordinates.
